# MiniGame002 v1.5.9 设计文档

## 版本主题
战斗距离修复 - 解决玩家无法攻击怪物的问题

---

## 1. 优化需求来源

根据优化方向列表第2条：战斗体验 - 遇怪处理优化

问题反馈：主角遇怪停止在攻击范围外，无法攻击怪，怪也打不到主角

---

## 2. 问题分析

### 2.1 当前问题
- 玩家 attackRange = 50像素 (5米)
- 怪物 stopDistance = 60像素 (6米) 
- 怪物在60像素外停止，玩家需要进入50像素才能攻击
- 导致玩家永远无法进入攻击范围，怪物也打不到玩家

### 2.2 影响
- 玩家无法击败怪物
- 游戏无法正常进行
- 战斗系统失效

---

## 3. 解决方案

### 3.1 方案一：调整玩家攻击范围（推荐）

将玩家 attackRange 从 50像素 调整为 70像素（7米）

**优点**：
- 玩家可以在怪物停止距离(60px)外进入攻击范围
- 与怪物攻击距离匹配
- 不需要修改怪物配置

### 3.2 方案二：调整怪物停止距离

将所有怪物的 stopDistance 调整为小于等于玩家 attackRange

**缺点**：
- 需要修改多个怪物配置
- 可能影响游戏平衡

---

## 4. 实施内容

### 4.1 代码修改

修改 `js/player.js`：

```javascript
// v1.5.9: 调整攻击范围至70px(7米)，确保能攻击到停在60px(6米)的怪物
attackRange: 70,  // 从50px调整为70px
```

### 4.2 验收标准

- [ ] 玩家能够进入怪物攻击范围并发起攻击
- [ ] 怪物死亡后玩家继续前进
- [ ] 战斗可以正常进行
- [ ] 无控制台错误
