# MiniGame002 v1.5.1 设计文档

## 版本主题
主角武器动画优化 - 攻击时武器动态效果

---

## 1. 优化需求来源

根据优化方向列表首位：主角攻击时剑要动起来（当前剑不动画）

---

## 2. 问题分析

### 2.1 当前问题
- 主角攻击时武器（剑）是静态的
- 武器没有跟随攻击动作进行摆动或挥动
- 降低了战斗打击感和视觉反馈
- 玩家难以判断是否正在攻击

### 2.2 影响
- 战斗体验单调，缺乏打击感
- 攻击动作不够生动
- 与Q版水墨风格斗游戏定位不匹配

---

## 3. 解决方案

### 3.1 武器摇摆动画

**实现方式**：
- 攻击时武器围绕角色手部位置进行周期性摆动
- 摆动角度：-15度到+15度
- 摆动频率：与攻击动作同步

**参数配置**：
```javascript
// weaponAnimation.js - 武器动画配置
const WEAPON_ANIMATION = {
    swingAngle: 15,        // 摇摆角度（度）
    swingSpeed: 8,         // 摇摆速度
    swingEase: 'ease-in-out'  // 缓动函数
};
```

### 3.2 攻击时的武器动态

**攻击触发时**：
- 武器从自然下垂位置开始
- 快速向前摆动至最大角度
- 然后回摆至另一侧
- 最后回到起始位置

**时间轴**：
| 阶段 | 时长 | 武器位置 |
|------|------|----------|
| 前摆 | 0.1秒 | 从下垂→前摆30度 |
| 挥动 | 0.15秒 | 前摆30度→后摆15度 |
| 回摆 | 0.1秒 | 后摆15度→回到原位 |

### 3.3 不同武器的动画效果

**剑**：
- 前后挥动式摆动
- 剑刃方向始终朝前

**刀**：
- 斜向劈砍式摆动
- 刀刃有旋转效果

**枪**：
- 前后戳刺式摆动
- 枪尖有震动效果

## 4. 技术实现

### 4.1 角色绘制修改

```javascript
// player.js - draw() 方法中添加武器动画
draw() {
    // ... 绘制身体部分 ...
    
    // 计算武器摇摆角度
    let weaponAngle = 0;
    if (this.attacking) {
        const t = (Date.now() - this.attackStartTime) / (this.attackInterval * 1000);
        weaponAngle = Math.sin(t * Math.PI * 2) * WEAPON_ANIMATION.swingAngle;
    }
    
    // 绘制武器（带动画角度）
    this.drawWeapon(weaponAngle);
}
```

### 4.2 武器绘制方法

```javascript
drawWeapon(angle) {
    ctx.save();
    ctx.translate(this.x + this.width/2, this.y + 10);  // 手部位置
    ctx.rotate(angle * Math.PI / 180);  // 应用摇摆角度
    
    // 根据武器类型绘制
    switch(this.weaponType) {
        case 'sword': this.drawSword(); break;
        case 'blade': this.drawBlade(); break;
        case 'spear': this.drawSpear(); break;
    }
    
    ctx.restore();
}
```
