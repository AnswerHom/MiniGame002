# MiniGame002 v1.3.2 设计文档

## 版本主题
怪物AI逻辑修复

---

## 1. 问题描述

### 1.1 问题现象
- 部分怪物的 `attackDistance`（攻击距离）小于 `stopDistance`（安全距离）
- 导致怪物走到安全距离后就停止，既不攻击也不继续靠近
- 玩家看到怪物"傻站"不动

### 1.2 问题原因
当前怪物配置中存在攻击距离小于安全距离的情况：
```
阴魂: attackDistance=40, stopDistance=60  ❌ 攻击距离 < 安全距离
妖狼: attackDistance=50, stopDistance=70  ❌ 攻击距离 < 安全距离
毒蛛: attackDistance=35, stopDistance=55  ❌ 攻击距离 < 安全距离
僵尸: attackDistance=45, stopDistance=65  ❌ 攻击距离 < 安全距离
```

---

## 2. 解决方案

### 2.1 修复规则
- **安全距离（stopDistance）不能大于攻击距离（attackDistance）**
- 正确逻辑：`attackDistance` ≥ `stopDistance`
- 怪物行为：
  1. 当距离 > stopDistance 时，向玩家移动
  2. 当 stopDistance ≤ 距离 < attackDistance 时，保持距离等待
  3. 当距离 < attackDistance 时，攻击玩家

### 2.2 修正怪物配置
| 怪物 | 原 attackDistance | 原 stopDistance | 修正后 attackDistance | 修正后 stopDistance |
|------|------------------|-----------------|----------------------|---------------------|
| 阴魂 | 40 | 60 | 60 | 60 |
| 妖狼 | 50 | 70 | 70 | 70 |
| 毒蛛 | 35 | 55 | 55 | 55 |
| 僵尸 | 45 | 65 | 65 | 65 |
| 蝴蝶精 | 30 | 50 | 50 | 50 |
| 毒蛇 | 25 | 45 | 45 | 45 |
| 骷髅 | 40 | 60 | 60 | 60 |
| 蝙蝠 | 35 | 55 | 55 | 55 |
| 魔藤 | 20 | 40 | 40 | 40 |
| 冰魔 | 50 | 75 | 75 | 75 |

### 2.3 代码逻辑优化
在 `Enemy.update()` 方法中，确保逻辑正确：
```javascript
// 优先检查攻击距离
if (absDist < this.attackDistance && this.attackCooldown <= 0) {
    this.attackPlayer();
    return;
}

// 检查安全距离（在攻击范围外保持距离）
if (absDist < this.stopDistance) {
    return; // 保持距离
}

// 距离足够远，继续移动
if (dist > 0) {
    this.x += this.speed * dt;
}
```
