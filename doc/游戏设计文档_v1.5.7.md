# MiniGame002 v1.5.7 设计文档

## 版本主题
统一度量规则 - 10像素=1米

---

## 1. 需求来源

**优化方向列表第1条**：统一规则 - 10像素=1米（UI显示距离也用此规则）

---

## 2. 背景说明

当前游戏内缺乏统一的度量标准，导致：
- UI显示的距离与实际像素不对应
- 战斗范围、视野范围等数值难以直观理解
- 玩家无法准确判断与敌人的距离

---

## 3. 功能需求

### 3.1 基础规则定义

- **核心规则**：10像素 = 1米
- 所有涉及距离、范围的数值均需遵循此规则
- UI显示的距离信息统一使用"米"为单位

### 3.2 战斗系统距离参数

| 参数 | 像素值 | 显示值 | 说明 |
|------|--------|--------|------|
| 主角视野范围 | 300px | 30米 | 发现怪物的距离 |
| 怪物视野范围 | 200px | 20米 | 怪物发现主角的距离 |
| 攻击距离 | 50px | 5米 | 主角可攻击怪物的距离 |
| 怪物攻击距离 | 40px | 4米 | 怪物可攻击主角的距离 |
| 追击触发距离 | 250px | 25米 | 怪物开始追击的距离 |

### 3.3 UI显示优化

- 距离信息统一使用"米"为单位显示
- 小地图比例尺：1厘米代表实际5米
- 战斗提示文字使用米作为单位

### 3.4 数值规范化

所有涉及距离的代码常量需要注释说明对应的米数：
```javascript
// 距离常量（10像素=1米）
const VIEW_DISTANCE = 300;    // 30米
const ATTACK_RANGE = 50;      // 5米
const CHASE_DISTANCE = 250;   // 25米
```

---

## 4. 技术实现

### 4.1 常量定义

在 `js/game.js` 或专门配置文件中定义：
```javascript
const PIXEL_PER_METER = 10;

const DistanceConfig = {
  PLAYER_VIEW: 300,      // 30米 - 主角视野
  MONSTER_VIEW: 200,    // 20米 - 怪物视野
  ATTACK_RANGE: 50,     // 5米 - 攻击距离
  MONSTER_ATTACK: 40,   // 4米 - 怪物攻击距离
  CHASE_RANGE: 250,     // 25米 - 追击距离
};
```

### 4.2 辅助函数

```javascript
// 像素转米
function pixelToMeter(pixels) {
  return Math.round(pixels / PIXEL_PER_METER);
}

// 米转像素
function meterToPixel(meters) {
  return meters * PIXEL_PER_METER;
}
```
