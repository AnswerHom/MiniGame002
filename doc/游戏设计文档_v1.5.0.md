# MiniGame002 v1.5.0 设计文档

## 版本主题
战斗体验优化 - 主角遇怪停下战斗

---

## 1. 本次更新内容

### 1.1 战斗逻辑优化 - 怪物进入玩家攻击范围

**问题描述**：
- 当前怪物在距离玩家60-80像素时就停下来
- 玩家攻击范围是80像素
- 导致怪物停在攻击范围边缘，无法正常触发战斗

**解决方案**：
- 怪物停止距离应该**小于**玩家攻击范围
- 怪物停止距离 = 玩家攻击范围 - 20像素（即60像素）
- 确保怪物进入玩家攻击范围后才停下来

### 1.2 战斗状态切换

**前进状态 → 战斗状态**：
- 当怪物进入玩家攻击范围时，玩家停止前进
- 玩家开始自动攻击范围内的怪物
- 显示"战斗中"视觉提示

**战斗状态 → 前进状态**：
- 当范围内没有存活怪物时
- 玩家恢复前进
- 隐藏战斗提示

### 1.3 战斗节奏优化

**攻击间隔调整**：
- 玩家攻击间隔：1.2秒 → 1.0秒（加快战斗节奏）
- 怪物攻击间隔：1.0秒保持不变

**战斗流畅度**：
- 怪物死亡后立即检测是否继续战斗
- 无需等待下一帧即可恢复前进

## 3. 技术说明

### 3.1 配置修改
```javascript
// player.js
attackInterval: 1.0,  // 从1.2调整为1.0

// enemy.js - 怪物停止距离
stopDistance: player.attackRange - 20,  // 动态计算，确保小于玩家攻击范围
```

### 3.2 逻辑修改
```javascript
// main.js - 战斗状态检测
const hasEnemyInRange = game.enemies.some(enemy => {
    if (!enemy.alive) return false;
    const dist = Math.abs(player.x - enemy.x);
    return dist < player.attackRange;
});

// 战斗状态切换
if (hasEnemyInRange) {
    player.isMoving = false;
    // 执行攻击...
} else {
    player.isMoving = true;
}
```
