---

## 2. 解决方案

### 2.1 安全距离机制

#### 问题分析
- 怪物会一直走到攻击距离范围内，与主角完全重叠
- 视觉上怪物与主角贴在一起，体验不佳

#### 实现方案
- 引入 `stopDistance`（安全距离/停止距离）概念
- 怪物在距离玩家 `stopDistance` 时停止移动
- 只有进入 `attackDistance` 范围时才攻击玩家
- 安全距离 > 攻击距离，确保怪物与主角保持一定距离

#### 配置示例
```
阴魂: { attackDistance: 40, stopDistance: 60 }
妖狼: { attackDistance: 50, stopDistance: 70 }
毒蛛: { attackDistance: 35, stopDistance: 55 }
僵尸: { attackDistance: 45, stopDistance: 65 }
```

## 3. 技术实现

### 3.1 Enemy类更新
- 新增 `stopDistance` 属性
- 修改 `update()` 方法：
  - 优先检查攻击距离（攻击玩家）
  - 其次检查安全距离（保持距离）
  - 最后才移动（接近玩家）

### 3.2 新增绘制方法

- `drawSnake()` - 毒蛇
- `drawSkeleton()` - 骷髅
