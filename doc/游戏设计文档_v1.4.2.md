# MiniGame002 v1.4.2 设计文档

## 版本主题
怪物数值平衡与成长曲线优化

---

## 1. 优化需求来源

### 用户反馈（2026-03-05）
- **优化怪物数值**：增加怪物属性成长曲线，不要一上来怪就那么难打

---

## 2. 问题分析

### 2.1 当前问题
- 怪物初始属性固定，前期玩家刚起步时怪物难度过高
- 缺少怪物随游戏进度的成长曲线
- 玩家体验：开局怪物太难打，劝退感强

### 2.2 期望效果
- 前期怪物较弱，让玩家轻松度过开局
- 随着游戏推进，怪物逐渐变强，保持挑战性
- 玩家成长速度与怪物成长速度保持合理平衡

---

## 3. 解决方案

### 3.1 怪物属性成长曲线

#### 3.1.1 成长系数设计
- **基础公式**：`怪物属性 = 基础值 × (1 + 成长系数 × 进度)`
- **进度计算**：`进度 = 已移动距离 / 1000`（每1000像素为1级）

#### 3.1.2 各怪物成长参数
| 怪物 | 基础HP | 成长系数 | 基础攻击 | 成长系数 |
|------|--------|----------|----------|----------|
| 阴魂 | 15 | 0.05 | 5 | 0.03 |
| 妖狼 | 25 | 0.06 | 8 | 0.04 |
| 毒蛛 | 20 | 0.05 | 10 | 0.03 |
| 僵尸 | 35 | 0.07 | 12 | 0.04 |
| 蝴蝶精 | 12 | 0.04 | 4 | 0.02 |
| 毒蛇 | 20 | 0.05 | 12 | 0.04 |
| 骷髅 | 40 | 0.08 | 14 | 0.05 |
| 蝙蝠 | 15 | 0.04 | 6 | 0.03 |
| 魔藤 | 30 | 0.06 | 15 | 0.05 |
| 冰魔 | 45 | 0.09 | 10 | 0.04 |

#### 3.1.3 成长曲线示例
- **阴魂（0进度）**：HP=15, 攻击=5
- **阴魂（10进度）**：HP=15×(1+0.05×10)=22.5 → 22, 攻击=5×(1+0.03×5 → 6
- **阴魂（20进度）**：10)=6.HP=15×(1+0.05×20)=30, 攻击=5×(1+0.03×20)=8

### 3.2 境界难度加成

不同境界的怪物额外属性加成：
| 境界 | HP加成 | 攻击加成 |
|------|--------|----------|
| 练气 | ×1.0 | ×1.0 |
| 筑基 | ×1.3 | ×1.2 |
| 金丹 | ×1.6 | ×1.4 |

## 4. 技术实现

### 4.1 修改Enemy类
```javascript
// 在Enemy构造函数中计算成长属性
constructor(x, type, realm = '练气') {
    const config = ENEMY_TYPES[type];
    const progress = player ? player.x / 1000 : 0;  // 根据玩家位置计算进度
    
    // 应用成长曲线
    this.hp = Math.floor(config.hp * (1 + config.hpGrowth * progress));
    this.attack = Math.floor(config.attack * (1 + config.attackGrowth * progress));
    
    // 应用境界加成
    const realmBonus = { '练气': {hp:1.0, atk:1.0}, '筑基': {hp:1.3, atk:1.2}, '金丹': {hp:1.6, atk:1.4} };
    const bonus = realmBonus[realm] || realmBonus['练气'];
    this.hp = Math.floor(this.hp * bonus.hp);
    this.attack = Math.floor(this.attack * bonus.atk);
    
    this.maxHp = this.hp;  // 记录最大HP用于血条显示
}
```

```javascript
const ENEMY_TYPES = {

    // ...
};
```
